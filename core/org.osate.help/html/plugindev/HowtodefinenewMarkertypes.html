8.5 How to define new Marker types
New markers can be defined as part of an OSATE plug-in plugin.xml specification.
<extension
id="YourAadlObjectMarker"
name="YourPluginObjectMarker"
point="org.eclipse.core.resources.markers">
<super
type="edu.cmu.sei.aadl.model.AadlObjectMarker">
</super>
<persistent
value="true">
</persistent>
</extension>
This new marker type can then be used by an ErrorReporter by simply creating the ErrorReporter with a second parameter, the marker type as string value. The string value is the fully qualified ID of the marker, i.e., the plug-in ID plus the marker ID.
You then use this error reporter as parameter to your plug-in switch constructor call in your OSATE plug-in action.
