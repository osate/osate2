Error-Model Annex v2

Introduction

The support of the Error-Model Annex is done by several means:index.html

  • Functional Hazard Assessment support for producing reliability report of your architecture.
  • Fault Impact traces the error path from error sources to the final component impact by the fault.
  • Unhandled faults report traces and logs potential modeling problems related to the error description of the system architecture.
  • Consistency Checks reports any discrepancy and inconsistency between the different error notations (component error model, composite error model, etc.).
  • Fault-Tree Analysis support for fault tree analysis from the AADL model. This is a tutorial illustrating its use through multiple steps in the development process. See also an SEI Blog post on the graphical EMFTA editor.
  • Instance Model Viewer support for viewing error and their propagations in the underlying architecture.
  • Reliability Block Diagram: provides metrics about the overall reliability of the system, based on a gross estimation.

These functionalities are available in the Fault Analysis menu from OSATE, as shown in the following picture. Note that most of functions apply to the instance model. To use them, you will need to select the system instance and run the expected analysis.
